# Sol v3.26.0 — The Build Session
*June 11 2026 · Build session with Claude Sonnet 4.6*

## What shipped today

### Infrastructure
- **Git repository initialised** — Sol app is now version-controlled. 5 commits. Nothing gets lost again.
- **app.json cleaned** — Removed dead biometric permissions (`USE_BIOMETRIC`, `USE_FINGERPRINT`) and `expo-local-authentication` plugin. These were leftover from a previous auth flow and would have raised questions at Play Store review.
- **Supabase graceful fallback** — `lib/supabase.ts` no longer crashes or silently fails when env vars are absent. All share functions return a clear "The Field is not yet open" message until real credentials are wired.

### School tab
- **Free tier daily cap** — Free users get 3 dives/day. On the 4th attempt: "The School rests after 3 dives. Return tomorrow — or unlock Sovereign study." Routes to paywall.
- **Watch-ad flow** — At the daily cap, users can watch a rewarded AdMob clip for +1 dive. Max 2 ads/day already coded. Sovereign never sees this.
- **Weekly Synthesis letter** — The weekly letter (generated from last 7 dives) now renders on school home as a collapsible `✉ WEEKLY SYNTHESIS` card. State + load logic already existed; JSX was missing.
- **Pattern notice banner** — Sol's pattern notice (triggered at ≥5 dives over ≥7 days) now renders as a dismissible banner above the domain grid. Same — logic existed, render was missing.

### Sanctum tab
- **Vigil in Today section** — Active vigil now shows in Sanctum Today with domain glyph, `Day X of 7` label, and a domain-color progress bar. Previously only visible in the School tab.
- **Today's school dives** — A `⊚ TODAY IN THE SCHOOL` strip shows all subjects studied today (with domain, glyph, duration in minutes).
- **Field profile erase button** — The FIELD PROFILE card now has an Erase button. Destructive confirm required. Clears tracked depth preferences, domain affinities, and top domains. Dive log and messages unaffected.
- **Time-aware intention label** — "MORNING INTENTION" now reads correctly as AFTERNOON / EVENING / NIGHT based on the hour. No more writing morning intentions at midnight.
- **Section tab counts** — `JOURNAL` and `VAULT` tabs now show live counts: `JOURNAL (12)`, `VAULT (3)`. You know at a glance what's there.
- **Journal + Vault empty states** — Both now speak in Sol's voice with context-appropriate instructions instead of generic filler text.
- **Field section intro** — Brief description added to the Field tab so new users understand what they're looking at.

### Companion (Sol home)
- **4 moods from usage data** — The companion glyph now computes its mood on focus:
  - `Dormant` — no chat in 3+ days → dim, slow pulse
  - `Present` — active → steady glow
  - `Lit` — 5+ dives this week → bounce
  - `Transcendent` — avg LQ ≥0.85 → breathe + accent color
- **Tap to speak** — Tap the companion glyph for a mood-appropriate phrase. 3-second display. No reproach for absence, ever (CLAUDE.md §XXVI).

### Library tab
- **Tab renames** — `CASCADE → SCORE`, `CEMENTER → CEMENT` — less jargon, clearer purpose.
- **Commons redesigned** — No longer a dead end. Shows 3 preview cards of what's coming (Shared Forge entries, Cement blocks, Open Seat threads). "YOUR PLACE IS KEPT."
- **SAVED empty state** — Sol-voiced with clear CTA: "The library holds what survives the test."
- **SCORE description** — Distinguishes it from the Forge (CASCADE scoring only vs. full pipeline).

### Dead code removed
- `_handleLongPressLegacy` — 145-line Alert.alert-based long-press handler deleted from `index.tsx`. The bottom-sheet modal (UX-03) replaced it weeks ago. Dead code was just sitting there.
